# Journal UI Guidelines (NewTIFI)

## Goals
- Present rigorous, scholarly content with immediate clarity.
- Make peer-reviewed content discoverable without paywalls.
- Maintain consistent journal identity across all pages.

## Global UI Rules
- **Typography**: Use clear hierarchy (H1 title, H2 section, H3 subsections).
- **Metadata**: Always include DOI, journal title, date, and author info.
- **CTA priority**: PDF/Document download and preview visible above the fold.
- **Consistency**: Journal pages share identical layout and UI structure.
- **Accessibility**: All actionable elements must be keyboard accessible.

## Journal Landing Page Structure
1. **Journal Identity Block**
   - Journal title
   - Scope statement (1–2 lines)
   - Cover/hero image (journal-specific)
   - ISSN, publisher, frequency
2. **Primary Navigation**
   - Journals | Reviews | Articles | Books | Interviews | Podcasts
3. **Article List**
   - Abstract preview cards with keywords
   - Expand/collapse abstract
   - Download/preview buttons
4. **Technical Info**
   - Peer review timeline
   - Licensing
   - Archiving/Indexing
5. **Submission Guidelines**
   - Rules, templates, and contact

## Article Page Structure
1. **Hero Area**
   - Journal label and DOI
   - Article title + abstract
2. **Author & Metadata Panel**
   - Author name + affiliation
   - Publication date + DOI
   - Keywords
3. **Academic Context**
   - Expanded abstract + research context paragraph
4. **Access Controls**
   - Preview (PDF only)
   - Download (PDF/DOCX)
5. **Journal Info Sidebar**
   - Journal title, ISSN, publisher, frequency

## Visual Standards
- **Journal images**: Use category-relevant imagery (finance, legal, digital).
- **Color usage**: NewTIFI navy + teal accents for hierarchy.
- **Card structure**: Use rounded containers and consistent padding.

## Content Rules
- **Abstract length**: 150–250 words.
- **Keywords**: 3–12, all caps if possible.
- **Citations**: DOI always visible and consistent format.
